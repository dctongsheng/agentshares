# BNB 自动交易系统设计方案

## 目标

在 BSC 链上构建一个简单的 BNB 自动交易系统，核心策略为**大跌买入、大涨卖出**（均值回归 / 逆向交易）。

## 当前上下文 / 假设

- **交易对**: BNB / USDT（或 BNB / BUSD）
- **链**: BSC (BNB Chain)
- **执行方式**: 链上 DEX swap（通过 OKX Onchain OS 的 `swap` 命令，或直接调用 PancakeSwap / Uniswap V3 合约）
- **已有工具**: OKX Onchain OS CLI（已安装 skills: `okx-dex-signal`, `okx-dex-trenches`，可能还有 `okx-dex-market`, `okx-dex-token`, `okx-dex-swap`, `okx-agentic-wallet`）
- **运行环境**: 可部署为 Python 定时脚本（cron job 或后台常驻进程）
- **资金规模**: 假设小资金试水，后续可扩展

## 策略设计

### 核心逻辑：逆向均值回归

```
价格短时大幅下跌 → 判断为超卖 → 买入
价格短时大幅上涨 → 判断为超买 → 卖出
```

### 1. 价格监控与信号判断

**数据源选择：**

| 方案 | 说明 | 推荐 |
|------|------|------|
| OKX Onchain OS `market kline` | 通过 CLI 获取 K 线数据 | ★★★ 首选，已安装 |
| PancakeSwap SDK | 直接读取链上池子价格 | 备选 |
| CoinGecko / DexScreener API | 免费价格 API | 备选 |

**"大跌 / 大涨" 的量化定义（多维度确认）：**

- **维度 A — 短时跌幅/涨幅阈值**
  - 监控周期: 1H / 4H K 线
  - 大跌判定: 1H 跌幅 > 3% 或 4H 跌幅 > 5%
  - 大涨判定: 1H 涨幅 > 3% 或 4H 涨幅 > 5%
  - 以上阈值可配置，初始保守，后续根据回测调整

- **维度 B — RSI 超卖/超买**
  - 计算 14 周期 RSI
  - 买入信号: RSI < 30（超卖区）
  - 卖出信号: RSI > 70（超买区）

- **维度 C — 成交量确认（可选增强）**
  - 跌幅伴随成交量放大 > 1.5 倍平均量 → 更可靠的恐慌性抛售信号
  - 涨幅伴随成交量放大 → 更可靠的 FOMO 信号

**信号组合规则（建议分阶段）：**

- **V1 简单版**: 只用维度 A（短时跌幅/涨幅阈值），降低复杂度
- **V2 增强版**: 维度 A + B 组合（跌幅/涨幅阈值 AND RSI 确认）
- **V3 完整版**: 维度 A + B + C 全部满足

### 2. 交易执行

**买入流程：**
```
检测到大跌信号
  → 检查钱包 USDT 余额是否足够
  → 计算买入数量（按仓位比例）
  → 执行 swap: USDT → BNB
  → 记录交易日志
```

**卖出流程：**
```
检测到大涨信号
  → 检查钱包 BNB 余额
  → 计算卖出数量（按仓位比例）
  → 执行 swap: BNB → USDT
  → 记录交易日志
```

**仓位管理（重要！）：**

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 单次买入比例 | 总资金的 10%-20% | 不 all-in，分批建仓 |
| 单次卖出比例 | 持有 BNB 的 20%-30% | 分批止盈 |
| 最大持仓比例 | 总资金的 60% | 控制风险敞口 |
| 最小持仓 USDT | 总资金的 20% | 保留子弹 |

**冷却期：**
- 每次交易后冷却 30-60 分钟，避免频繁交易被套

### 3. 风控机制

- **止损**: 单次交易亏损 > 5% 时强制止损卖出
- **日最大交易次数**: 限制每天最多 6 次交易
- **日最大亏损**: 累计亏损 > 3% 时当天停止交易
- **异常检测**: 价格源异常（API 超时、数据为 0）时不交易
- **Gas 保护**: BNB 余额保留 0.01 BNB 用于 Gas

### 4. 信号增强（利用已安装的 OKX Skills）

- **聪明钱信号（okx-dex-signal）**: 大跌时检查聪明钱是否也在买入 → 增强信心
- **KOL 动向**: 如果 KOL 也在买，可以适当加仓
- **不使用 trenches**: BNB 不是 meme coin，不需要 trenches 分析

## 分步实施计划

### Phase 1: 基础框架（Day 1）

1. **创建项目结构** — 初始化 `bnb-trader/` 目录和上述文件
2. **配置文件** — 编写 `config.yaml`，包含所有可调参数
3. **价格监控模块** — 使用 OKX CLI `onchainos market kline` 获取 BNB 价格
4. **RSI 计算** — 实现 14 周期 RSI 指标计算
5. **验证**: 手动运行价格获取，确认数据准确

### Phase 2: 策略引擎（Day 2）

1. **信号判断逻辑** — 实现维度 A（跌幅/涨幅阈值检测）
2. **仓位计算** — 根据配置的仓位比例计算买卖数量
3. **冷却期机制** — 记录上次交易时间，执行冷却逻辑
4. **验证**: 用历史 K 线数据回测信号，调整阈值参数

### Phase 3: 交易执行（Day 3）

1. **钱包查询** — 使用 OKX CLI 查询 BNB / USDT 余额
2. **Swap 执行** — 使用 OKX CLI `onchainos swap execute` 或直接调用 PancakeSwap Router
3. **交易日志** — 每笔交易记录时间、方向、价格、数量、原因
4. **验证**: 小额测试交易（$10 级别）

### Phase 4: 风控 + 通知（Day 4）

1. **止损逻辑** — 检查持仓浮亏，触发止损
2. **日频限制** — 交易次数和累计亏损控制
3. **异常处理** — API 超时、网络错误、余额不足等
4. **消息通知** — 可选接入 Telegram Bot 推送交易通知
5. **验证**: 模拟异常场景测试风控触发

### Phase 5: 部署上线（Day 5）

1. **部署方式选择**:
   - 方案 A: Hermes cron job（简单，定时轮询）
   - 方案 B: 后台常驻 Python 进程（实时，推荐）
   - 方案 C: 服务器 + systemd service（长期稳定）
2. **日志轮转** — 防止日志文件过大
3. **监控看板**（可选）— 简单 HTML 页面展示交易历史和持仓
4. **验证**: 先用模拟盘运行 24 小时，确认无异常后切换实盘

## 策略参数初始建议

```yaml
# config.yaml
trading:
  chain: bsc
  token_in: USDT
  token_out: BNB
  trade_amount_pct: 0.15        # 单次买入占总资金 15%
  sell_amount_pct: 0.25          # 单次卖出持仓的 25%
  max_position_pct: 0.60         # 最大持仓 60%
  min_reserve_pct: 0.20          # 最低 USDT 保留 20%
  gas_reserve_bnb: 0.01          # Gas 保留
  cooldown_minutes: 45           # 交易冷却期
  max_daily_trades: 6            # 日最大交易次数
  max_daily_loss_pct: 0.03       # 日最大亏损 3%

strategy:
  # 维度 A — 价格阈值
  check_interval_seconds: 300    # 5 分钟检查一次
  drop_threshold_1h: 0.03        # 1H 跌幅 > 3%
  drop_threshold_4h: 0.05        # 4H 跌幅 > 5%
  rise_threshold_1h: 0.03        # 1H 涨幅 > 3%
  rise_threshold_4h: 0.05        # 4H 涨幅 > 5%

  # 维度 B — RSI
  rsi_period: 14
  rsi_oversold: 30               # RSI < 30 买入
  rsi_overbought: 70             # RSI > 70 卖出

  # 止损
  stop_loss_pct: 0.05            # 单次止损 5%

  # 信号增强
  check_smart_money: true        # 检查聪明钱信号

logging:
  file: data/trades.json
  level: INFO
```

## 风险与注意事项

| 风险 | 说明 | 应对 |
|------|------|------|
| **策略失效** | 趋势性行情中逆向策略会持续亏损 | 加入趋势过滤器（MA200），趋势明确时不逆向 |
| **滑点** | 链上 swap 滑点可能较大 | 设置滑点容忍度（1-2%），大额交易拆分 |
| **API 限流** | OKX CLI 可能有调用频率限制 | 控制轮询频率，必要时加缓存 |
| **私钥安全** | 自动交易需要私钥/钱包授权 | 使用专用交易钱包，仅放交易资金 |
| **网络延迟** | BSC 节点偶尔拥堵 | 多 RPC 备用节点 |
| **合约风险** | DEX 合约潜在风险 | 使用主流 DEX（PancakeSwap） |

## 总结

这是一个**中等复杂度**的链上自动交易项目。核心难度不在策略本身（逆向均值回归是经典策略），而在于：

1. **可靠的信号检测** — 数据获取稳定性
2. **安全的交易执行** — 链上 swap 的滑点和 Gas 管理
3. **严格的风控** — 防止单边行情中持续亏损

建议从 V1（纯价格阈值）开始，小额跑 1-2 周验证效果，再逐步加入 RSI 和聪明钱信号增强。
